# 数据结构教材算法模板拓展路线图

> 基于严蔚敏《数据结构》（C 语言版）及其配套习题集，系统梳理 Cide 代码模板体系下一阶段需要补充的算法与数据结构模板。所有模板均要求使用**标准 C 实现**，不因 IDE 编译器限制而扭曲算法原貌。

---

## 一、当前模板覆盖现状

截至 2026-06-06，`CideFlutter/lib/models/code_template.dart` 中已内置 **43 个模板**，覆盖：

- **排序**：冒泡、选择、插入、快速、归并、堆排序、希尔排序、计数排序
- **查找**：线性查找、二分查找
- **图算法**：BFS、DFS
- **动态规划**：斐波那契 DP、01 背包
- **数据结构**：顺序表、链表（节点/头插/尾插/遍历/删除/双向链表/循环链表）、栈/链栈、队列/链队列/循环队列、二叉树（节点/先序/中序/后序/层序/BST 插入与查找）、哈希表
- **字符串**：字符串反转
- **基础/递归**：数组遍历、指针交换、GCD、素数判断、约瑟夫环、阶乘、斐波那契递归、汉诺塔

## 二、下一阶段模板全景清单

按教材章节与算法重要性分为三级：

- **P0**：教材最核心、课堂必讲、考研必考的算法，建议优先实现。
- **P1**：教材标准例题或经典应用，教学价值高，建议第二批补充。
- **P2**：拓展内容、竞赛向或代码量较大的高级结构，有余力再实现。

## 四、模板实现规范

所有新增模板必须遵守以下规范，确保不因 IDE 缺陷扭曲算法：

### 4.1 算法原貌保持

- **不允许**为绕过 IDE 限制而改变算法标准流程（例如：为了避开二级指针就把头插法改成返回值传参，除非教材本身也采用返回值风格）。
- **允许**使用教材中常见的等价写法（例如：哈夫曼树用静态数组模拟优先队列，而不是调用标准库 `qsort` 或手写堆，因为教材通常采用前者）。
- **允许**使用指针返回结构体，或函数返回指针，以替代按值返回大结构体（这是 C 语言中完全标准的惯用写法）。

### 4.2 代码模板格式

所有模板必须定义在 `CideFlutter/lib/models/code_template.dart` 的 `CodeTemplate.defaults` 列表中，格式如下：

```dart
CodeTemplate(
  'templateKey',
  '模板显示名称',
  '分类',
  '#include <stdio.h>\n'
  '... 代码字符串 ...\n',
  params: [
    TemplateParam(key: 'n', label: '参数说明', defaultValue: '5', type: ParamType.int),
  ],
  tutorialSteps: [
    TutorialStep(
      title: '步骤标题',
      description: '步骤描述',
      focusLines: [4, 5, 6],
      explanations: [
        LineExplanation(line: 4, short: '短标题', detail: '详细说明'),
      ],
    ),
  ],
),
```

占位符使用 `{{key:defaultValue}}` 语法，参数校验规则沿用现有逻辑：

- `ParamType.int`：非负整数，上限 1000。
- `ParamType.string`：任意字符串。
- `ParamType.identifier`：标识符格式。

### 4.3 教程步骤要求

- **P0 模板必须带 `TutorialStep`**，每个模板至少 3 个步骤。
- **P1/P2 模板建议带教程**，若代码极简单可省略。
- 每个 `TutorialStep` 必须包含 `focusLines`（要高亮的代码行号）和至少一个 `LineExplanation`。
- 行号从 1 开始，与代码模板中实际行号一致。

### 4.4 编译器特性确认

P0 模板依赖的 C 特性当前均已支持，无需新增编译器功能：

| 特性 | 状态 | 说明 |
|------|------|------|
| 结构体定义与成员访问 | ✅ 支持 | `struct Node { ... };`、`.` / `->` |
| 指针与二级指针 | ✅ 支持 | `struct Node**`、`LinkList*` |
| 函数返回结构体 | ✅ 支持 | 按值返回结构体已修复 |
| 函数返回指针 | ✅ 支持 | `Node* createNode(...)` |
| `malloc` / `free` | ✅ 支持 | 含泄漏检测与 UAF/DoubleFree 检测 |
| `qsort` + 函数指针 | ✅ 支持 | Kruskal 所需 |
| 数组与多维数组 | ✅ 支持 | 邻接矩阵、哈夫曼数组 |
| 递归 | ✅ 支持 | AVL、BST、DFS、归并已验证 |
| `for` / `while` / `if` / `return` | ✅ 支持 | 基础控制流 |
| 字符串函数 | ✅ 支持 | `strlen`、`strcmp`、`strcpy` |
| `const` / `typedef` / `enum` | ✅ 支持 | 类型系统 |

## 五、与现有系统的联动

新增模板后，需要同步更新以下位置：

1. **`AGENTS.md`**
   - 更新"代码模板扩展"段落的模板总数。
   - 更新模板分类列表，新增本次支持的算法。

2. **`native/src/diagnostics/learning_path.rs`**
   - 若新增模板对应某种常见认知误区（如 KMP 对应字符串理解、AVL 对应树旋转），可在 `recommend_learning_paths` 中引用新模板的 `target_id`。

3. **`docs/current/ALGORITHM_DATASTRUCTURE_DESIGN.md`**
   - 若新增模板涉及算法检测器（如拓扑排序、最短路径）的语义标签支持，需要同步更新算法识别清单。

4. **测试覆盖**
   - 每个新增模板至少对应一个 E2E 测试或影子验证用例，确保代码能在 Cide 编译器中正确编译运行并输出预期结果。

## 六、任务拆分建议

### 批次 1：P0 核心（13 个模板）

预计工作量：中等（每个模板约 30~60 行 Dart 代码，含教程步骤）。

推荐实现顺序：

1. 循环链表、静态链表（线性表收尾）
2. KMP、BF 模式匹配（串章节闭环）
3. 并查集（为 Kruskal 做准备）
4. 线索二叉树、哈夫曼树、AVL 树（树章节闭环）
5. Prim、Kruskal、Dijkstra、Floyd、拓扑排序（图章节闭环）
6. 基数排序（排序章节闭环）

### 批次 2：P1 拓展（约 18 个模板）

包括：括号匹配、表达式求值、串基本操作、`nextval`、BST 删除、有序表合并、插值/斐波那契查找、关键路径、LCS/LIS/硬币找零/矩阵连乘、桶排序、埃氏筛等。

### 批次 3：P2 高级（约 8 个模板）

包括：双端队列、多项式相加、树/森林转换、红黑树、B 树、Bellman-Ford、SPFA、外部排序等。代码量较大或教材涉及较少，按需实现。

## 七、验收标准

每个模板完成后必须通过以下检查：

1. **编译通过**：能在 Cide 编译器中零错误编译。
2. **运行正确**：输出与标准 C 编译器（Clang/GCC）执行结果一致。
3. **参数替换正确**：`buildCode()` 能正确替换 `{{key:default}}` 占位符。
4. **教程高亮正确**：`focusLines` 行号与代码实际行号一致，`LineExplanation` 无错别字。
5. **不扭曲算法**：代码逻辑与教材标准实现一致，没有为绕过 IDE 限制而引入的非常规写法。
