### 3.1 前端 P0 Bug

**状态**: ✅ 已完成（见 6.1 #1、#2）

#### Bug 1：`LineNumberColumn` 变量访问后缀永远不显示

**文件**: `CideFlutter/lib/editor/gutter/line_number_column.dart:36`

```dart
final markers = ctx.accessedVars
    .where((a) => false) // 由外层统一计算当前行的变量标记
    .take(2)
    ...
```

`where((a) => false)` 导致结果恒为空，变量访问标记功能实际不可用。

#### Bug 2：`applyFix` 基于中文字符串匹配

**文件**: `CideFlutter/lib/providers/ide_notifier.dart:512-639`

Rust 后端已经返回完整结构化修复字段（`fix_kind`、`replace_start_line`、`replace_start_column`、`replace_end_line`、`replace_end_column`、`replacement_text`），但 Dart 端：

1. 结构化替换**只支持单行**（`if (startLine == endLine)`），多行替换直接跳过
2. fallback 使用中文/符号字符串匹配（`fix.contains('分号')`、`fix.contains("}'")` 等），文案微调即失效

**建议**: 将修复逻辑完全下沉到 Rust，新增 `rust.applyFix(filename, source, diag) -> String?` API。

### 3.4 `cargo fmt` 实际不可用

**状态**: ✅ 已解决（见 6.2 P1-7）

`rustfmt.toml` 原配置了 4 个 nightly-only 特性：

```toml
group_imports = "StdExternalCrate"
imports_granularity = "Module"
wrap_comments = true
format_code_in_doc_comments = true
```

CI 和本地都使用 stable toolchain，导致这些特性不生效且每次 `cargo fmt` 打印警告。`cargo fmt --check` 还会检测出 4 处格式差异：

- `native/src/bin/cide_cli.rs:293`
- `native/src/capi/mod.rs:303`
- `native/src/compiler/algorithm_detector.rs`（换行风格）
- `native/src/compiler/ast.rs:578, 871`

**建议**: CI 改用 nightly toolchain 运行 `cargo fmt --check`，或移除 nightly-only 配置。
