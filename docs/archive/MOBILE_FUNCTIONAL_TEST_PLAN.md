## 🔧 adb 问题排查指南

### Step 1: 确认 adb 能否找到设备

在 PowerShell 中执行：

```powershell
adb devices
```

**可能的结果及处理：**

| 输出 | 状态 | 解决措施 |
|------|------|---------|
| `xxxxxxxx    device` | ✅ 正常 | 可直接运行 `.	est-mobile.ps1 -Install -Run` |
| `xxxxxxxx    offline` | ⚠️ 掉线 | 执行 `adb kill-server` 然后 `adb start-server`，保持手机屏幕亮屏 |
| `xxxxxxxx    unauthorized` | ❌ 未授权 | 看手机屏幕是否有"允许 USB 调试"弹窗，**必须点击确定** |
| **空白无任何输出** | ❌ 未识别 | 见 Step 2 |

### Step 2: 空白/无设备识别的处理

1. **换数据线**：很多数据线仅支持充电，**换一根确定能传数据的线**
2. **换 USB 口**：插到机箱**后置 USB 口**（供电更稳定）
3. **开启开发者选项**：
   - 设置 → 关于手机 → 连续点击"版本号"7 次 → 开启开发者模式
   - 设置 → 系统/更多设置 → 开发者选项 → **开启"USB 调试"**
4. **保持亮屏**：锁屏/休眠会导致 adb 断开，开启开发者选项里的"不锁定屏幕"
5. **处理弹窗**：重新插拔手机，手机屏幕出现"允许 USB 调试吗？"弹窗，必须点击**确定/允许**

### Step 3: 脚本内置 adb 路径是否正确

项目脚本 `test-mobile.ps1` 会按以下顺序查找 adb：

1. `PATH` 环境变量中的 `adb`
2. `D:\Program Files (x86)\Microsoft Visual Studio\Shared\Android\android-sdk\platform-tools\adb.exe`

如果你的 adb 在其他位置，可以临时指定：

```powershell
$env:Path += ";C:\Your\Path\To\platform-tools"
.\test-mobile.ps1 -Install -Run -Logcat
```

### Step 4: 手动安装测试（绕过脚本）

如果脚本一直失败，可以手动执行：

```powershell
# 1. 构建 APK（跳过 Native .so 编译）
.\test-mobile.ps1 -SkipNativeBuild

# 2. 手动安装
adb install -r "dist/android/com.cide.mobile-Signed.apk"

# 3. 手动启动
adb shell monkey -p com.cide.mobile -c android.intent.category.LAUNCHER 1

# 4. 查看日志（调试用）
adb logcat --pid=$(adb shell pidof com.cide.mobile)
```

## 📋 测试执行建议

### 最小测试集（P0 必测）

首次真机验证时，建议按以下顺序执行最小测试集：

1. **启动测试**：应用能正常启动，不闪退
2. **编辑测试**：能输入代码，虚拟键盘不遮挡
3. **编译测试**：运行示例代码，输出正确结果
4. **错误测试**：输入错误代码，能看到红色错误行和诊断信息
5. **单步测试**：点击单步，能看到黄色高亮行移动
6. **FAB 测试**：展开菜单，打开调试 Modal
7. **内存测试**：单步执行时内存面板有数据更新
8. **旋转测试**：横竖屏切换不崩溃

### 使用脚本自动化

```powershell
# 完整流水线：构建 → 安装 → 启动 → 实时日志
.\test-mobile.ps1 -Install -Run -Logcat

# 前端代码改动后快速重测（跳过 Native .so 编译）
.\test-mobile.ps1 -SkipNativeBuild -Install -Run

# Release 模式最终验证
.\test-mobile.ps1 -Configuration Release -Install -Run
```
